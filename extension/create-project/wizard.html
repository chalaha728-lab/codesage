    Create Project — settings

    GitHub token

    Needs ONLY repo + workflow scopes. Do NOT check all permissions — that's a security risk. For fine-grained PATs, grant account-level Administration: Read + Contents: Read and write + Workflows: Read and write.

    checking…

    Get one at github.com/settings/tokens/new (classic) or github.com/settings/personal-access-tokens/new (fine-grained).

    Appetize.io API key

    Stored locally + pushable to GitHub repos as a secret so the workflow uploads APKs to Appetize automatically — no large downloads on your machine.

    checking…

    Get one at appetize.io/settings. Click "Add on GitHub" to open your repo's secrets page. Add a new secret: Name = APPETIZE_API_KEY, Value = your Appetize token.

    GitHub MCP server

    Configures the GitHub MCP server (from the marketplace) to use your stored GitHub token instead of OAuth. This fixes the "incompatible auth server: does not support dynamic client registration" error. The AI can then read/edit files on GitHub directly via MCP tools.

    checking…

    After configuring, reload the VS Code window (Ctrl/Cmd+Shift+P → "Developer: Reload Window") for the MCP server to pick up the new config.
